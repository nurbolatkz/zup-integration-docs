1. Назначение сервиса

Сервис предназначен для двунаправленного обмена кадровыми данными между внешними системами (CRM, корпоративные порталы, кадровые системы) и конфигурацией Зарплата и управление персоналом для Казахстана (1С:ЗУП, ред. 3.1).

Через сервис планируется обеспечить:

  • Чтение нормативно-справочных данных: сотрудники, физические лица, подразделения, должности, лицевые счета, графики работы и другие;
  • Создание кадровых документов: приём на работу, кадровый перевод, изменение оплаты труда, больничный лист, отпуск по уходу за ребёнком, увольнение;
  • Обновление ранее созданных кадровых документов.

2. Краткое описание методов

Планируемый HTTP-сервис предоставит следующие конечные точки:

HTTP-методURL-шаблонОперацияНазначение
GET/dt/storage/{Storage}/{ID}ЧтениеПолучить объект по типу и идентификатору
POST/dt/storage/{Storage}/{ID}СозданиеСоздать новый объект
PUT/dt/upload/{ID}ОбновлениеОбновить существующий объект по ID
GET/dt/download/{ID}СкачиваниеСкачать файловый объект
GET/dt/volume/{VolumeID}/*Файл по томуПолучить файл по тому и пути
POST/dt/volume/{VolumeID}/*Загрузка файлаЗагрузить файл

Параметр {Storage} определяет тип объекта (справочник или документ). Значения приведены в разделах 3–4.

3. Справочники (чтение)

Справочники доступны только для чтения через GET /dt/storage/{Storage}/{ID}. Идентификатор объекта передаётся в параметре {ID} как string.

4. Документы (создание / обновление)

Кадровые документы создаются через POST /dt/storage/{Storage}/{ID} и обновляются через PUT /dt/upload/{ID}. Тело запроса — JSON. Колонка Обяз. указывает, является ли поле обязательным при создании.

6. Обработка ошибок

При ошибке сервис возвращает JSON следующего вида:

json{
  "success": false,
  "errorCode": "ValidationError",
  "message": "Не заполнено обязательное поле: employeeId",
  "details": ""
}
ObjectNotFound
Объект с указанным ID не найден
→ Проверить корректность string
InvalidStorage
Неверный тип хранилища (Storage)
→ Проверить имя объекта из спецификации
ValidationError
Отсутствуют обязательные поля
→ Добавить недостающие поля и повторить
DuplicateDocument
Документ на период + сотрудник уже существует
→ Использовать PUT для обновления
InternalError
Внутренняя ошибка 1С
→ Сообщить администратору интеграции
Unauthorized
Ошибка авторизации (HTTP 401)
→ Проверить учётные данные

8. Пользовательская документация

8.1. Когда читать справочники

Данные справочников следует считывать перед созданием кадровых документов для получения актуальных идентификаторов (string) объектов. Рекомендуется:

  • кэшировать редко изменяемые справочники (Должности, Подразделения, Начисления, Графики работы);
  • обновлять кэш по расписанию (например, ежедневно) или при обнаружении ошибки ObjectNotFound.

8.2. Порядок создания документа

  1. Получить string нужных объектов через GET-запросы к справочникам.
  2. Сформировать тело JSON-запроса, заполнив все обязательные поля.
  3. Отправить POST-запрос на /dt/storage/{Storage}/00000000-0000-0000-0000-000000000000 (нулевой string при создании).
  4. Проверить поле success в ответе.
  5. Сохранить возвращённый id документа для возможного последующего обновления через PUT.

8.3. Правила обработки ошибок

СитуацияДействие
success = trueДокумент создан / обновлён. Сохранить id.
ValidationErrorДобавить недостающие обязательные поля и повторить запрос.
ObjectNotFoundПроверить корректность string объектов. Обновить кэш справочников.
DuplicateDocumentДокумент уже существует — использовать PUT для обновления.
InternalErrorЗафиксировать текст ошибки и передать администратору интеграции.
HTTP 401Проверить логин/пароль или токен авторизации.

8.4. Контрольные правила

  • Не создавать документы без предварительной проверки наличия сотрудника (ObjectNotFound при GET).
  • Хранить id созданных документов в журнале обмена для последующей сверки.
  • При повторной передаче данных за тот же период сверять id: разный id означает создание нового документа.
  • Даты передавать строго в формате yyyy-MM-dd (ISO 8601).
  • Все string передавать в нижнем регистре с дефисами: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.
